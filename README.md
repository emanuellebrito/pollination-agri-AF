# Spatial mismatch between pollination demand and pollinator habitat supply in Brazil's Atlantic Forest

This repository contains the reproducible analysis workflow for a manuscript assessing where pollination-dependent agriculture in the Brazilian Atlantic Forest is spatially exposed to low habitat-based potential pollinator supply.

The workflow combines municipal crop production data, crop-specific pollination-dependence scores, native vegetation and agricultural land-cover information from MapBiomas, and a 5-km spatial grid to estimate pollination demand, potential habitat supply, pollination deficit risk, demand-supply mismatch, critical hotspots, economic exposure, and crop-level drivers.

## Repository structure

Only code is stored in this GitHub repository. Large input data files are archived externally and should be downloaded separately.

```text
.
├── 01_prepare_pam_grid_inputs.R
├── 02_GEE_calculate_mapbiomas_grid_areas.txt
├── 03_main_analysis.R
├── data_raw/                  # not tracked on GitHub; added after downloading data archive
└── output_mata_atlantica/      # generated outputs; not tracked on GitHub
```

## Scripts

### `01_prepare_pam_grid_inputs.R`

Prepares the spatial and agricultural input data for the analysis. This script:

- reads the Atlantic Forest biome boundary;
- reads Atlantic Forest municipalities;
- reads PAM crop production data for 2020–2024;
- joins crop records to the pollination-dependence reference table;
- creates municipal pollination-demand files;
- creates the 5-km Atlantic Forest grid;
- exports intermediate files to `output_mata_atlantica/`.

Main outputs include:

```text
output_mata_atlantica/atlantic_forest_grid_5km.gpkg
output_mata_atlantica/atlantic_forest_state_boundaries_2020.gpkg
output_mata_atlantica/pam_atlantic_forest_long_2020_2024_dependence.csv
output_mata_atlantica/municipal_pollination_demand_mean_2020_2024.csv
output_mata_atlantica/municipal_pollination_demand_2024_for_gee.csv
```

### `02_GEE_calculate_mapbiomas_grid_areas.txt`

Google Earth Engine script used to calculate agricultural and native vegetation area from MapBiomas 2024 for each grid x municipality intersection.

This step exports:

```text
mapbiomas_grid_atlantic_forest_2024.csv
```

After exporting the CSV from Google Earth Engine, place it in:

```text
data_raw/mapbiomas_grid_atlantic_forest_2024.csv
```

If the Google Earth Engine assets are stored under a different account, update the asset paths inside the script before running it.

### `03_main_analysis.R`

Runs the main analysis. This script:

- reads the canonical inputs generated by script 01 and the Google Earth Engine step;
- calculates municipal pollination demand;
- calculates total agricultural production value (EV), animal pollination economic value (APEV), and relative vulnerability (RV);
- redistributes municipal values to 5-km grid cells using agricultural area weights;
- estimates habitat-based potential pollinator supply;
- calculates pollination deficit risk and demand-supply mismatch;
- identifies critical hotspots, defined as high demand and low supply cells;
- summarizes hotspot concentration, state-level exposure, crop drivers, and correlations;
- exports final tables and spatial outputs.

Main outputs include:

```text
output_mata_atlantica/grid_pollination_mismatch_mean_2020_2024.gpkg
output_mata_atlantica/grid_pollination_mismatch_crop_metrics_mean_2020_2024.gpkg
output_mata_atlantica/grid_pollination_mismatch_crop_metrics_mean_2020_2024.csv
output_mata_atlantica/critical_hotspots_mean_2020_2024.csv
output_mata_atlantica/critical_hotspots_by_state_mean_2020_2024.csv
output_mata_atlantica/economic_value_annual_totals_2020_2024.csv
output_mata_atlantica/tables/Table_1_summary_general.csv
```

## How to run

1. Clone or download this repository.

2. Download the external data archive:

```text
[ADD FIGSHARE OR ZENODO LINK HERE]
```

3. Create a `data_raw/` folder in the repository root and place all required input files inside it.

4. Open R or RStudio and set the working directory to the repository root.

5. Run the first R script:

```r
source("01_prepare_pam_grid_inputs.R")
```

6. Run the Google Earth Engine script in the Earth Engine Code Editor:

```text
02_GEE_calculate_mapbiomas_grid_areas.txt
```

Download the exported CSV and save it as:

```text
data_raw/mapbiomas_grid_atlantic_forest_2024.csv
```

7. Run the main analysis:

```r
source("03_main_analysis.R")
```

The main outputs will be written to:

```text
output_mata_atlantica/
```

## Notes on reproducibility

- The GitHub repository stores only scripts.
- The folder `data_raw/` is intentionally excluded from GitHub because it contains large input data files.
- The folder `output_mata_atlantica/` is generated by the scripts and does not need to be versioned.
- The Google Earth Engine step is required only if the MapBiomas grid-level CSV is not already included in the external data archive.
- If running the scripts on another computer, check that all paths point to the repository root and that no user-specific `setwd()` path remains in the scripts.
